# FIX_UI.md — что подкрутить в UI редактора

## 🚨 СРОЧНАЯ ПРАВКА (регрессия после прошлого прохода) — читать первой
**Симптом:** фон-«стол» стал ТЁМНЫМ вместо светлого. Стеклянные панели из-за этого
выглядят грязно-серыми/металлическими, подписи-лейблы (ШРИФТ, ЦВЕТ ТЕКСТА, ВЫРАВНИВАНИЕ)
почти не читаются. Раньше (до прошлого прохода) фон был светлее и приятнее.

**Причина:** приложение LiteEditorAI само по себе тёмное (тёмная тема). В прошлый проход
добавили цветные blob-пятна, но НЕ задали под них светлую непрозрачную подложку — поэтому
пятна легли на тёмный фон приложения. А полупрозрачные панели `rgba(255,255,255,0.5)` поверх
тёмного фона дают серую муть вместо матового белого стекла.

**Дизайн редактора — СВЕТЛЫЙ** (в отличие от остального тёмного приложения). Это норма: модуль
редактора должен принудительно перекрывать тёмную тему своим светлым фоном.

**Что сделать:**
1. Корневому контейнеру модуля редактора («стол», самый внешний div на `100vw×100vh`) задать
   НЕПРОЗРАЧНЫЙ светлый градиент как базу:
   `background: linear-gradient(140deg,#eef1f7,#e7ecf5 60%,#eaf3f0);`
   Именно на этот контейнер, чтобы перекрыть тёмный фон приложения.
2. Слой blob-пятен — ПОВЕРХ этой светлой базы (z-index выше базы, ниже контента). Значения
   пятен см. в разделе «Приоритет 1» ниже (они правильные, менять не надо — им просто нужна
   светлая подложка снизу).
3. Панели — поверх blob-слоя. Тогда `rgba(255,255,255,0.5)` + `backdrop-filter: blur()`
   наконец дадут светлое матовое стекло, а не серую муть.
4. Проверить, что тёмная тема приложения не переопределяет фон/цвет текста внутри модуля через
   токены (`var(--bg)`, `var(--text)` и т.п.) или более специфичные селекторы. Возможно, для
   контейнера редактора надо локально переопределить эти токены на «светлые» значения.
5. Текст и лейблы внутри светлых панелей должны быть ТЁМНЫМИ: основной `#1c1c1e`,
   лейблы `#8a8a8e`. Если тёмная тема делает текст светлым — переопределить внутри модуля.
6. Проверить, что предыдущие правки (карточка «Формула», анимация переключателей) не потерялись.

**Итог:** после этого фон должен стать светлым бледно-голубым, панели — белым матовым стеклом,
пятна — мягкими цветными бликами. Сверить с эталоном `Lite Editor v2 (автономный).html`.

## Приоритет 1 — Фон-«стол» бледный, пропали цветные «blob»-пятна
**Проблема:** вокруг белой страницы фон почти белый и плоский. В эталоне фон живой:
светлый градиент + четыре размытых цветных пятна дают глубину.

**Как в эталоне (из дизайн-README и прототипа):**
- Градиент подложки: `linear-gradient(140deg,#eef1f7,#e7ecf5 60%,#eaf3f0)`.
- Четыре «blob»-пятна (абсолютное позиционирование, `border-radius:50%`,
  `pointer-events:none`, лежат в слое под контентом):
  1. оранжевое `#ffd8a8` — `width/height:640px; top:-180px; left:-140px; filter:blur(36px); opacity:.75`
  2. фиолетовое `#d0bfff` — `560px; top:-120px; right:-120px; filter:blur(40px); opacity:.7`
  3. голубое `#99e9f2` — `600px; bottom:-220px; left:22%; filter:blur(44px); opacity:.6`
  4. розовое `#ffc9d6` — `420px; bottom:-160px; right:8%; filter:blur(40px); opacity:.55`
  (каждое — `radial-gradient(circle, <цвет>, transparent 68%)`).

**Проверить:** не перекрывает ли пятна непрозрачный фон контейнера модуля; слой пятен
должен быть НИЖЕ контента, но ВЫШЕ базового фона. Возможно, у контейнера редактора стоит
сплошной `background`, который надо убрать/сделать прозрачным.

## Приоритет 2 — Слабый эффект матового стекла (frosted glass)
**Проблема:** панели (title bar, инспектор, карточки «Формат»/«Формула», пилюля
форматирования) выглядят почти сплошными белыми. В эталоне они полупрозрачные —
сквозь них просвечивает цветной фон с размытием.

**Как в эталоне:**
- Фон панели: `rgba(255,255,255,0.5)` … `0.55` (НЕ сплошной белый!).
- `backdrop-filter: blur(24–30px) saturate(180–185%)` + вендорный
  `-webkit-backdrop-filter: …` (в Electron/Chromium нужен именно webkit-префикс).
- Бордер: `1px solid rgba(255,255,255,0.65–0.7)`.
- Тень: `0 10px 30px rgba(30,40,80,0.13)` + внутренний блик `inset 0 1px 0 rgba(255,255,255,0.9)`.

**Частые причины, почему стекло «не видно»:**
1. Фон панели задан сплошным (`#fff` / `opacity:1`) вместо `rgba(...,0.5)` — тогда размывать
   нечего. Сделать фон полупрозрачным.
2. Нет `-webkit-backdrop-filter` (Chromium его требует) — добавить рядом с `backdrop-filter`.
3. За панелью нет цветного фона (см. Приоритет 1) — стекло видно только когда под ним есть цвет.

## Приоритет 1 — Не показывается панель (карточка) «Формула»
**Проблема:** во вкладке «Редактирование» инспектора нет карточки «Формула»
(есть только «Формат»). В эталоне там две карточки: «Формат» и «Формула».

**Что проверить:**
1. Разметка карточки «Формула» вообще есть в `renderer/module.html`? Если нет — добавить
   (в прототипе `Редактор v2 (финал).dc.html` это блок `data-drop="formula"`: сетка
   быстрых символов 8×3, `<textarea>` для LaTeX, блок «Предпросмотр», кнопка
   «Вставить в документ»).
2. В прототипе карточка обёрнута в условие `showFormula` (проп, по умолчанию `true`).
   Проверить, что это условие не выключено и карточка не скрыта через `display:none`.
3. Логика формул завязана на KaTeX. Если KaTeX ещё не подключён локально (см. CLAUDE.md,
   подводный камень №1), предпросмотр не отрисуется — но сама карточка всё равно должна
   быть видна. Сверить обе вещи.
4. Набор быстрых символов (из прототипа, массив `SYMBOLS`): x², x₂, ½, √, ∑, ∫, ∏, lim,
   π, α, β, θ, ≤, ≥, ≠, ±, ×, ÷, →, ∞, ā (вектор), ∂, ∈, · — каждый вставляет свой
   LaTeX-сниппет в textarea.

## Приоритет 2 — Переключатели должны «переезжать» (анимация)
**Проблема:** сегментные переключатели (Claude/Codex, Разметка/Markdown, вкладки
AI-обработка/Редактирование, роли) сейчас просто мгновенно меняют активную кнопку.
Хочется, чтобы белый «бегунок» плавно **переезжал** к выбранному пункту.

**Как сделать (паттерн «sliding thumb»):**
- Сейчас активность рисуется сменой фона у самой кнопки (`background:#fff` на активной).
  Для анимации переезда так не выйдет — фон просто перекрашивается на месте.
- Правильный приём: внутри контейнера сегмента добавить отдельный элемент-«бегунок»
  (абсолютно позиционированная белая пилюля-подложка под текстом кнопок), а кнопки
  сделать прозрачными. Бегунок двигать через `transform: translateX(...)` и менять
  `width` под активную кнопку.
- На бегунок повесить `transition: transform .22s cubic-bezier(.4,0,.2,1), width .22s cubic-bezier(.4,0,.2,1);`
  — тогда он плавно едет.
- Позицию/ширину бегунка вычислять от активной кнопки (её `offsetLeft`/`offsetWidth`)
  при каждом переключении. Контейнеру задать `position:relative`, бегунку — `z-index` ниже текста.
- Цвет активного текста (акцент) можно анимировать через `transition: color .18s`.
- Применить ко всем сегментам единообразно: модель (Claude/Codex), режим
  (Разметка/Markdown), вкладки инспектора. Для чипов ролей достаточно плавной смены
  фона/бордера (`transition: all .15s`) — там «переезд» не нужен, они в несколько рядов.
